## 我们的目标

<Callout type="success">
在输入几行命令后，**成功在终端返回 Neovim 的版本号**
</Callout>

## 开始之前

- [ ] 已安装 [Scoop](../../toolchain/package-managers/scoop/install)（Windows） 或 [Homebrew](../../toolchain/package-managers/homebrew/install)（macOS）

## 动手操作

### 1. 检查是否已安装

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
nvim --version
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
nvim --version
```
</Tab>

</Tabs>

如果之前没有成功安装过，终端将会返回类似下面的输出，让我们继续下一步：

```
nvim : 无法将“nvim”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。请检查名称的拼写，如果包括路径，请确保路径正确
，然后再试一次。
所在位置 行:1 字符: 1
+ nvim --version
+ ~~~~
    + CategoryInfo          : ObjectNotFound: (nvim:String) [], CommandNotFoundException
    + FullyQualifiedErrorId : CommandNotFoundException
```

### 2. 安装 Neovim

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
scoop install neovim
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
brew install neovim
```
</Tab>

</Tabs>

### 3. 验证版本

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
nvim --version
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
nvim --version
```
</Tab>

</Tabs>

终端将会返回版本号：

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
NVIM v0.12.1
Build type: Release
LuaJIT 2.1.1774638290
Run "nvim -V1 -v" for more info
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
NVIM v0.12.1
Build type: Release
LuaJIT 2.1.1774896198
Run "nvim -V1 -v" for more info
```

</Tab>

</Tabs>
