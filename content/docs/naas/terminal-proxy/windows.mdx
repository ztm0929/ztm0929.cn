---
title: "如何开启终端代理"
---

<DraftNotice />

很多小白打开了系统代理，却发现终端里的下载、安装、更新命令还是慢。原因通常不是网络本身，而是终端会话里没有持续生效的代理环境变量。我们一起来开启「终端代理」，让后续常见命令长期稳定加速。

## 我们的目标

<Callout type="success">
运行少量命令，为终端成功注入代理，**使 [`git`](/docs/toolchain/git)、[`pip`](/docs/toolchain/python/pip) 等命令运行速度提升**
</Callout>

## 开始之前

- [ ] 已开启 [系统代理](../../system-proxy/clash-verge-rev)
- [ ] 系统自带 Windows Terminal（Windows 终端）或 Terminal.app（macOS 终端）
- [ ] 具备基本的终端操作能力（打开终端、复制粘贴命令、<kbd>Enter</kbd> 回车确认）

## 动手操作

### 1. 环境预检

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
$env:HTTP_PROXY; $env:HTTPS_PROXY;
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
echo $HTTP_PROXY; echo $HTTPS_PROXY;
```

</Tab>

</Tabs>

### 2. 注入环境变量

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
"http_proxy","https_proxy","HTTP_PROXY","HTTPS_PROXY" | % { [System.Environment]::SetEnvironmentVariable($_,"http://127.0.0.1:7897","User") }
```

</Tab>

<Tab value="macOS">

```zsh title="Terminal.app（终端）"
echo "export http_proxy=http://127.0.0.1:7897" >> ~/.zshrc
echo "export https_proxy=http://127.0.0.1:7897" >> ~/.zshrc
echo "export all_proxy=socks5://127.0.0.1:7897" >> ~/.zshrc

echo "export HTTP_PROXY=$http_proxy" >> ~/.zshrc
echo "export HTTPS_PROXY=$https_proxy" >> ~/.zshrc
echo "export ALL_PROXY=$all_proxy" >> ~/.zshrc
```

</Tab>

</Tabs>


### 3. 确认注入成功

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
$env:HTTP_PROXY; $env:HTTPS_PROXY;
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
echo $HTTP_PROXY; echo $HTTPS_PROXY;
```

</Tab>

</Tabs>

### 4. 测试实际代理效果（可选）

当终端返回 `http://127.0.0.1:7897` 的结果时，说明已成功注入，我们可以测试一下代理效果，这一步骤不是必需的。 

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
curl.exe ipinfo.io/city
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
curl ipinfo.io/city
```

</Tab>

</Tabs>

可以看看 [ipinfo.io/city](https://ipinfo.io/city) 返回的结果，它应该是代理服务器所在的城市，而不是我们当前所在的城市。

## 常见问题

<Accordions>
<Accordion title="为什么终端显示已有代理，但还是很慢甚至无法连接？">
终端代理的前提是已启用系统代理，请检查 [小猫咪](../../system-proxy/clash-verge-rev) 是否正常运行。
</Accordion>
<Accordion title="为什么之前配置过并且生效了，有一天又不行了？">
本文演示的是 Windows PowerShell/PowerShell 两款 Shell 注入环境变量的操作步骤，Git Bash、CMD 或其他的 Shell 需要额外注入。
</Accordion>
</Accordions>

## 继续探索

完成目标之后，休息一下。我们可以尝试这些：

- 为 git、pip 等程序单独配置代理
- 通过 `Get-ChildItem Env:` 命令，探索除了 HTTP(S)_PROXY 之外的 更多 Windows PowerShell 环境变量
