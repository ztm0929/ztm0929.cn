---
title: "如何为 Windows 终端启用代理"
---

很多小白打开了系统代理，却发现终端里的下载、安装、更新命令还是慢。原因通常不是网络本身，而是终端会话里没有持续生效的代理环境变量。本文会带你用最少步骤，把 Windows 下的终端代理配置成“开终端就可用”，让后续常见命令长期稳定加速。

## 我们的目标

**git/pip 等命令运行速度提升**

## 开始之前

- [x] 已安装 Windows Terminal（终端）

## 动手操作

<div className='fd-steps [&_h3]:fd-step'>

```powershell title="Windows Terminal（终端）"
"http_proxy","https_proxy","HTTP_PROXY","HTTPS_PROXY" | % { [System.Environment]::SetEnvironmentVariable($_,"http://127.0.0.1:7897","User") }
```

### 确认注入成功

```powershell title="Windows Terminal（终端）"
$env:HTTP_PROXY; $env:HTTPS_PROXY;
```

### 测试代理效果（可选）

当终端返回 `http://127.0.0.1:7897` 的结果时，说明已成功注入，我们可以测试一下代理效果，这一步骤不是必需的。 

```powershell title="Windows Terminal（终端）"
curl.exe ipinfo.io
```

## 常见问题

<Accordions>
<Accordion title="为什么终端显示已有代理，但还是很慢甚至无法连接？">
终端代理的前提是已启用系统代理，请检查 [小猫咪](../../system-proxy/clash-verge-rev) 是否正常运行。
</Accordion>
<Accordion title="为什么之前配置过并且生效了，有一天又不行了？">
本文演示的是 Windows PowerShell/PowerShell 两款 Shell 注入环境变量的操作步骤，Git Bash、CMD 或其他的 Shell 需要额外注入。
</Accordion>
</Accordions>
