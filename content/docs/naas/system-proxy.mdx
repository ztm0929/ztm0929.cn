- [GitHub](https://github.com) 几乎是编程学习的必经之地，其中有大量优秀开发者的代码可供学习
- [Hugging Face](https://huggingface.co) 是 AI 领域的资源宝库
- 而 [YouTube](https://www.youtube.com) 则有许多编程教学视频

我们在访问这类网站查找学习资料时，常见的体验是加载很慢、图片刷不出来、甚至频繁超时——很多时候并不是电脑或浏览器的问题，而是网络“走的路”不够顺。

所谓「系统代理」，可以简单理解为：

> 让系统的网络请求先交给一个代理工具/远程服务器进行转发，再去访问目标网站，相当于把出网通道统一切到一条更合适的路径上。

开启它的意义在于提升连接的稳定性与成功率，让日常学习与查资料的访问更连贯；

接下来只需要做少量设置，并用几个网站和 IP 信息做一次验证，就能确认系统代理已经生效。

Clash Verge Rev 是一款知名的网络代理工具，可以理解成「系统代理的中控台」（我常叫作“小猫咪”）：

**它把原本分散在系统设置里的代理开关、订阅/节点管理、规则分流、DNS 与连接日志等能力集中到一个界面里统一管理。**

这样做的好处是：需要优化网络时，不用到处改设置、也不容易漏掉关键选项；只要在这里完成导入配置、选择节点、打开系统代理，并通过日志/连通性验证确认“真的生效”，日常访问学习资料就会更稳定、更省心。

接下来的操作基本都会围绕它展开。

## 我们的目标

<Callout type="success">

完成本教程后，我们将**开启系统代理并确认已生效**：

1. GitHub、Hugging Face、YouTube 至少能稳定打开其首页并加载主要内容
2. 访问 [IPinfo](https://ipinfo.io/city) 显示的城市与未开启代理时不同
3. 在 Clash Verge Rev 的连接/日志中能看到对应网站的请求记录

</Callout>

## 动手操作

### 1. 环境预检

我们先别急着配置，先用 1 分钟做个小测试，看看当前网络的表现：

依次打开：

- GitHub：[github.com](https://github.com)
- Hugging Face：[huggingface.co](https://huggingface.co)
- YouTube：[youtube.com](https://www.youtube.com)

观察页面能否稳定打开，主要内容（文字/图片等）能否正常加载。

再打开 [IPinfo](https://ipinfo.io/city) ，把显示的城市名称记下来。

<Callout type="warning">
如果 Hugging Face / YouTube 打不开、很慢、或者一直转圈，这是很常见的现象

把“打不开的样子”和 ipinfo 的结果先记下来就好，接下来开启系统代理后再回来对比，很容易看到变化
</Callout>

点击下面的卡片，下载「小猫咪」在对应平台的安装包：

### 2. 导入订阅

- 在「设置」页面启用「DNS 覆写」并关闭「IPv6」
- 在「订阅」页面输入链接并点击「导入」，订阅链接输入私密信息，私信天明获取
- 导入成功后在「代理」页面应该能看到许多节点

### 3. 开启基础选项

- 系统代理
- 开机自启
- 静默启动

### 4. enjoy!

- 此时应该能够流畅访问 GitHub、HugingFace 等网站了
