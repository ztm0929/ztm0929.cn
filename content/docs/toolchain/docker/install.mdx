## 开始之前

<Callout>
由于 Docker 限制，Windows 和 macOS 系统需要额外安装 Docker Desktop，它不够轻便，因此本教程将以 Linux 系统为例进行演示

Windows 系统请使用 [WSL](../../shell-and-terminal/windows-terminal/wsl) 安装 Docker，macOS 系统请使用 [Colima](https://colima.run/) 取代 Docker
</Callout>



- [ ] Linux 发行版（如 Ubuntu、Debian、Fedora 等）

## 动手操作

### 1. 添加官方源 

```
sudo apt update
sudo apt install ca-certificates curl gnupg

sudo install -m 0755 -d /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

### 2. 安装 Docker Engine

```
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### 3. 开机自启

```
sudo systemctl start docker
sudo systemctl enable docker
sudo systemctl status docker
```

### 4. 权限

```
sudo usermod -aG docker $USER
newgrp docker
```

### 5. 网络代理

```
sudo nano /etc/docker/daemon.json
```

```
{
  "proxies": {
    "http-proxy": "http://192.168.1.7:7897",
    "https-proxy": "http://192.168.1.7:7897",
    "no-proxy": "localhost,127.0.0.1"
  }
}
```

### 6. 测试

```
docker run hello-world
```

## 总结回顾

我们主要做了以下几件事：

1. 添加 Docker 官方源，因为系统默认的软件源可能没有最新版本的 Docker
2. 安装 Docker Engine 以及相关组件
3. 设置 Docker 开机自启，确保系统重启后 Docker 服务能够自动启动
4. 配置用户权限，允许当前用户直接使用 Docker 命令而无需 sudo
5. 配置网络代理，确保 Docker 能够通过代理服务器访问互联网
