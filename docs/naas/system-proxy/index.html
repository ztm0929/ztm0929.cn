如何开启系统代理

使用「小猫咪」优化网络

注意

本文尚处于草稿状态，内容可能不完整或存在错误，如有疑问，欢迎 联系天明 改进！
  • GitHub 几乎是编程学习的必经之地，其中有大量优秀开发者的代码可供学习
  • Hugging Face 是 AI 领域的资源宝库
  • 而 YouTube 则有许多编程教学视频

我们在访问这类网站查找学习资料时，常见的体验是加载很慢、图片刷不出来、甚至频繁超时——很多时候并不是电脑或浏览器的问题，而是网络“走的路”不够顺。

所谓「系统代理」，可以简单理解为：

让系统的网络请求先交给一个代理工具/远程服务器进行转发，再去访问目标网站，相当于把出网通道统一切到一条更合适的路径上。

开启它的意义在于提升连接的稳定性与成功率，让日常学习与查资料的访问更连贯；

我们只需要做少量设置，并用几个网站和 IP 信息做一次验证，就能确认系统代理已经生效。

clash-verge-rev/clash-verge-rev

124.4K9.1K

Clash Verge Rev 是一款知名的网络代理工具，可以理解成「系统代理的中控台」（我常叫作“小猫咪”）：

它把原本分散在系统设置里的代理开关、订阅/节点管理、规则分流、DNS 与连接日志等能力集中到一个界面里统一管理。

这样做的好处是：需要优化网络时，不用到处改设置、也不容易漏掉关键选项；只要在这里完成导入配置、选择节点、打开系统代理，并通过日志/连通性验证确认“真的生效”，日常访问学习资料就会更稳定、更省心。

接下来的操作基本都会围绕它展开。

我们的目标

完成本教程后，我们将开启系统代理并确认已生效：

  1. GitHub、Hugging Face、YouTube 至少能稳定打开其首页并加载主要内容
  2. 访问 IPinfo 显示的城市与未开启代理时不同
  3. 在 Clash Verge Rev 的连接/日志中能看到对应网站的请求记录

动手操作

环境预检

我们先别急着配置，先用 1 分钟做个小测试，看看当前网络的表现：

依次打开：

  • GitHub：github.com
  • Hugging Face：huggingface.co
  • YouTube：youtube.com

观察页面能否稳定打开，主要内容（文字/图片等）能否正常加载。

再打开 IPinfo ，把显示的城市名称记下来。

如果 Hugging Face / YouTube 打不开、很慢、或者一直转圈，这是很常见的现象

把“打不开的样子”和 ipinfo 的结果先记下来就好，接下来开启系统代理后再回来对比，很容易看到变化

安装并打开「小猫咪」

点击下面的卡片，下载「小猫咪」在对应平台的安装包：

Windows 安装包

适用于 Windows 10/11，版本号 v2.5.1

macOS 安装包

适用于 macOS，版本号 v2.5.1

双击运行下载的安装包，按照提示完成安装并运行：

  1. 双击 Clash.Verge_2.5.1_x64-setup.exe 安装包开始安装（弹出「用户帐户控制」就点「是」）
  2. 安装语言选中文，安装路径保持默认，一路「下一步」
  3. 安装完成后启动 Clash Verge Rev
  4. 看到软件主界面（或任务栏托盘出现图标）就算安装成功
  1. 打开 Clash.Verge_2.5.1_aarch64.dmg，把图标拖进「应用程序」
  2. 在「应用程序」中打开 Clash Verge Rev
  3. 如果系统弹出安全提示，按提示允许打开
  4. 能正常打开软件主界面就算安装成功

导入订阅 拿到节点

打开「小猫咪」以后，先把订阅导入进来，让「代理」页面出现可用节点：

  1. 进入「设置」页面，开启「DNS 覆写」并关闭「IPv6」
  2. 进入「订阅」页面，粘贴订阅链接并点击「导入」
  3. 导入成功后，进入「代理」页面，应该能看到许多可选项，它们就是具体的节点

补充说明

  • 「DNS 覆写」和「IPv6」属于高级选项，按步骤打开/关闭即可
  • 「订阅链接」是一串私密 URL，请勿泄露，需要的话可私聊天明获取（微信或 Telegram）

开启系统代理

  1. 进入「代理」页面，选择一个绿色的节点（表示在线且可用）
  2. 进入「设置」页面，打开「系统代理」开关
  3. 建议同时打开「开机自启」和「静默启动」的开关，它们能让「小猫咪」的运行更省心

验证代理效果

现在回到第一步做同样的测试，对比一下结果：

  1. 打开 GitHub / Hugging Face / YouTube，观察是否能稳定加载主要内容
  2. 打开 IPinfo，看看城市是否与开启前不同
  3. 进入「小猫咪」的「连接」或「日志」页面，看看是否有对应网站的请求记录

做到上面 3 条，就说明系统代理已经生效了，日常访问学习资料的网络体验应该会有明显提升，enjoy!

总结回顾

总结一下，我们通过「小猫咪」成功优化了我们的网络体验：

  1. 了解系统代理的概念和意义
  2. 下载并安装「小猫咪」
  3. 导入订阅并选择可用节点
  4. 开启系统代理并验证效果

常见问题

如果在操作过程中遇到其他问题，可以先参考「小猫咪」的官方说明文档，或者直接联系天明寻求帮助。

继续探索

  • 「代理服务器」是一个专有技术名词，「小猫咪」帮我们简化了配置流程，具体详见微软或苹果的官方说明：在 Windows 中使用代理服务器 或 在 Mac 上输入代理服务器设置
  • 当前只是配置了系统代理，对于我们这种有命令行的需求的编程小白，还需要启用 终端代理 继续优化网络
