运行少量命令，**在终端成功返回 Git 的版本号**
</Callout>

## 开始之前

- [x] 良好的网络环境（已开启 [终端代理](../../../naas/terminal-proxy)）
- [x] 已安装 [Scoop](../../package-managers/scoop/install)（Windows）或 [Homebrew](../../package-managers/homebrew/install)（macOS）

## 动手操作

### 1. 检查是否内置 Git

有时操作系统会预装 Git，可以先检查一下：

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
git --version
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
git --version
```
</Tab>

</Tabs>

如果返回了 Git 的版本号，说明已经安装了 Git，可以跳过后续步骤。如果提示命令未找到，则继续安装。

### 2. 安装 Git

<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
scoop install git
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
brew install git
```

</Tab>

</Tabs>

### 3. 验证安装


<Tabs groupId="os" items={['Windows', 'macOS']} persist>

<Tab value="Windows">

```powershell title="Windows Terminal（终端）"
git --version
```

</Tab>

<Tab value="macOS">

```sh title="Terminal.app（终端）"
git --version
```

</Tab>

</Tabs>


如果成功安装，将显示 Git 的版本号。

## 总结回顾

回顾一下，我们需要做的主要就是三个步骤：

1. 检查是否内置 Git
2. 安装 Git（如果没有内置）
3. 验证安装成功

## 常见问题

网上已有的各类 Git 安装教程都是基于官方图形界面的，我们在 [启用了终端代理](../../../naas/terminal-proxy) 、[使用软件包管理器安装](../../package-managers)，也许不会遇到什么问题。

- `git init`：在当前目录初始化一个新的 Git 仓库
- `git add <file>`：将文件添加到暂存区
- `git commit -m "message"`：提交更改并添加提交信息
- `git status`：查看当前仓库的状态
