## 我们的目标

在完成基础信息填写后，**成功让脚本自动完成购票流程**，全程只打开浏览器一次。

## 开始之前

- [ ] 良好的网络环境（已开启 [终端代理](../naas/terminal-proxy) ）
- [ ] 已安装 [Git](../git) 和 [Python](../python/pip)

## 动手操作

### 1. 检查基本环境

```powershell title="Windows Terminal（终端）"
Write-Host "【代理环境变量】"; $env:http_proxy; $env:https_proxy; `
Write-Host "`n【Git 版本】"; git --version; `
Write-Host "`n【Python 版本】"; python --version; `
Write-Host "`n【pip 版本】"; pip --version
```

### 2. 获取源码并初始化

```powershell title="Windows Terminal（终端）"
git clone https://github.com/Matt-Dong123/tools4szu.git
```

项目源码较粗糙，没有 requirements.txt 这样的依赖文件，我们需要手动安装一下所需依赖：requests、pandas 和 openpyxl。

```powershell title="Windows Terminal（终端）"
cd tools4szu/venue-helper
python -m venv venv
venv\Scripts\activate
pip install requests pandas openpyxl
```

在正式开始前可以做个小检查，确认是否成功安装了依赖：

```powershell title="Windows Terminal（终端）"
pip list
```

## 常见问题

<Accordions>
<Accordion title="与浏览器自动化相比，脚本的特点是什么？">
由于不需要处理界面，只需要向目标接口发起网络请求，因此速度更快，效率更高。但行为特征较明显，服务器更容易识别为自动化操作。
</Accordion>
<Accordion title="为什么预约失败？">
Cookie 有效期较短（大约 3 到 4 小时），建议在抢票时间半小时前手动登录获取最新的 Cookie。
</Accordion>
</Accordions>
